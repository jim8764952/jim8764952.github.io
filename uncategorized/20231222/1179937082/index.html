Python底線的秘密：真的是讓人眼花撩亂

Posted on 2023-12-22 Edited on 2024-01-21 Views:

前言

這篇 是我覺得講解的很好的裡面還有提到好用的底線用法，以下我用自己的方式去理解底線的運作規則。

摘要

底線的變數通常就這四種。

  1. __ 變數__ : 已實作出的函數，官方建議不要自己發明，但還是能覆寫這方法。
  2. __變數 : 防止父子類別相互覆蓋變數。
  3. 變數_ : 與內建函數名稱同名時能做區分。
  4. _變數 : 建議不要取用內部的變數。

觀察 : 其實Python沒那麼嚴格，不會強制你不能做，只是要知道你做了會有甚麼後果。

__ 變數__

直接舉例__eq__、__ne__等 這些是已提供的方法，當然你還是可以去覆寫。

__變數

透過前雙底線的名字修飾機制，父類別跟子類別的變數即使命名相同，都透過「被 Python 直譯器擅自改名字」的方式通通保留下來了，不會互相覆蓋！**文章**

變數_

這個我就比較好了解，在寫題目時有一題問地圖上有幾座島嶼，那我想用 map 作命名，但內建函數已存在map()函數，這時就能在我的變數後方加上單個底線作使用 map_ 就保留了內建函數的功能。

_變數

在使用from mypackage import * 時 _變數 不會被匯入，但還是想用還是能呼叫。

單底線 _

這是我自己很常會用到的 _ 像是在宣告空陣列時，我不需要使用到 i 就能以 _ 代替。

另外在文章中看到的，在數值中間放置不影響輸出的數值。
