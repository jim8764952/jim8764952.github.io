PEP8 到底在幹嘛，PEP8 共通性的程式碼風格。

前言

在我產製作品時，還沒意識到我排版能力需要待加強，以往也是手動進行排版也沒有與人共同開發的經驗，倘若未來與人合作時這樣會花上許多不必要的心力，沒有排版的程式碼就像是看一個字跡瞭草沒有段落分明的作文，所以在產製告一個階段後，我除了安裝 Flake8 輔助我排版，也花了一些時間閱讀官網提供的排版指引 PEP8 ，在下面的小節我會點出重要的部分，提醒我就算沒有編輯器下也能符合 PEP8 風格。

變數命名

有關底線的命名是我不熟的地方，所以我另外寫一篇筆記來更深入去學習。

通常這樣命名

  1. 類別 : MyClass(cls) 單字開頭都大寫
  2. 函數 : myclass(self) 全小寫
  3. 常數 : PI = 3.14 全大寫

除了上述的撰寫風格，我覺得變數命名的用字更重要，一開始覺得用a、b、c這種命名法不是可以打很快嗎，但最後看都看不懂這些變數是存放甚麼數值的，所以我先從 github 上的專案學習高手怎麼命名，有些變數是大家都有共識的像是 self 或是模組官網有提供通用的縮寫舉例 : matplotlib.pyplot -> plt 。一方面我也要多看國外文章補充英文字彙量才能命出簡短又容易了解含意的名稱。
